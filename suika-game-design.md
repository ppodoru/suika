# 수박 게임(Suika Game) 웹 버전 설계 문서

이 문서는 닌텐도 스위치의 인기 게임인 '수박 게임(Suika Game)'을 웹 환경에서 최대한 동일하게 구현하기 위한 상세 설계 가이드입니다. (나무위키 최신 정보 반영)

---

## 1. 개요 (Overview)
*   **게임명:** 수박 게임 웹(Suika Game Web)
*   **장르:** 퍼즐, 물리 시뮬레이션
*   **목표:** 과일을 떨어뜨려 같은 종류끼리 합쳐 더 큰 과일을 만들고, 최종적으로 '수박'을 만드는 것. 과일이 상단 경계선을 넘지 않도록 관리하며 최대한 높은 점수를 획득해야 함.

---

## 2. 게임 메커니즘 (Game Mechanics)

### 2.1 과일 진화 시스템 (Evolution)
*   동일한 과일 두 개가 충돌하면 다음 단계의 과일 한 개로 합쳐짐(Merge).
*   **진화 순서 (총 11단계):**
    1. 체리 (Cherry)
    2. 딸기 (Strawberry)
    3. 포도 (Grape)
    4. 귤 (Tangerine)
    5. 감 (Persimmon)
    6. 사과 (Apple)
    7. 배 (Pear)
    8. 복숭아 (Peach)
    9. 파인애플 (Pineapple)
    10. 멜론 (Melon)
    11. 수박 (Watermelon)
*   **특수 규칙:** **수박과 수박이 만나면 소멸**하며 높은 점수를 획득함.

### 2.2 점수 시스템 (Scoring)
과일이 합쳐질 때마다 획득하는 점수 (사용자 정의 기준):
| 합쳐지는 과일 (결과물 아님) | 획득 점수 |
| :--- | :--- |
| 체리 + 체리 | 1 |
| 딸기 + 딸기 | 3 |
| 포도 + 포도 | 6 |
| 귤 + 귤 | 10 |
| 감 + 감 | 15 |
| 사과 + 사과 | 21 |
| 배 + 배 | 28 |
| 복숭아 + 복숭아 | 36 |
| 파인애플 + 파인애플 | 45 |
| 멜론 + 멜론 | 55 |
| 수박 + 수박 | 66 |

### 2.3 조작 및 생성 (Controls & Spawning)
*   **마우스/터치 이동:** 화면 상단에서 과일의 가로 위치를 조절.
*   **클릭/탭:** 과일을 수직으로 떨어뜨림.
*   **제한 사항:** 플레이어가 직접 떨어뜨릴 수 있는 과일은 **1단계(체리)부터 5단계(단감)**까지만 랜덤하게 결정됨.
*   **차례 대기:** 과일을 하나 떨어뜨린 후, 물리 연산이 어느 정도 안정화되거나 특정 시간 대기 후 다음 과일 생성.

### 2.4 물리 엔진 상세 (Physics)
*   **질량 동일의 법칙 (Critical):** **모든 과일의 질량(Mass)은 동일함.** 
    *   이로 인해 작은 체리가 큰 수박을 밀어내거나 위치를 조정하는 전략적 플레이가 가능함.
    *   대신 면적이 작은 과일이 큰 과일 사이의 틈새를 잘 파고들어 아래쪽에 쌓이는 현상이 발생함.
*   **콜라이더(Colliders):** 각 과일의 실제 외형에 맞춘 히트박스 사용.
    *   **비정형:** 딸기, 포도, 파인애플 등은 울퉁불퉁하거나 각진 형태를 반영하여 물리적 불규칙성 제공.
*   **강한 반동 (Pop Force):** 합체 시 두 과일의 좌표가 겹치며 발생하는 반동과 탄력이 매우 큼. 주변 과일들을 강하게 튕겨내며, 이로 인해 상단 선을 넘는 변수가 자주 발생.
*   **마찰력 및 탄성:** 마찰력은 낮고 탄성은 높게 설정하여 과일들이 잘 미끄러지고 역동적으로 움직이게 함.

### 2.5 게임 오버 (Game Over)
*   상단에 흰색 점선 형태의 **데드라인(Death Line)**이 존재.
*   바구니 안의 과일이 이 선에 닿거나 넘어가는 순간 **즉시 게임 종료**.
    *   단순히 쌓여 있는 상태뿐만 아니라, **합체 시 발생하는 반동(Pop)으로 인해 과일이 튀어 올라 선을 넘는 경우**도 포함됨.
*   **예외:** 플레이어가 조작 중인 과일은 판정에서 제외.

## 3. 기술 스택 및 구현 로직

*   **물리 엔진:** Matter.js (2D) - 질량 동일 설정(`mass: 1`) 및 `restitution`(탄성) 조정 필요.
*   **병합 로직:** `collisionStart` 이벤트에서 동일 라벨 확인 후 중심점에서 다음 단계 생성. 수박+수박의 경우 생성 없이 제거만 수행.
